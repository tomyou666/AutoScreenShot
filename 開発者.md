## 開発者向けメモ

このプロジェクトで開発・デバッグを始めるときの手順をまとめます。

### 1. 初回セットアップ

- **Go Modules 初期化（まだ `go.mod` が無い場合のみ）**

```bash
go mod init AutoScreenShot
go mod tidy
```

- **マニフェスト埋め込み用ツール `rsrc` のインストール（初回のみ）**

```bash
go install github.com/akavel/rsrc@latest
```

### 2. マニフェスト（Common Controls v6）の有効化

`walk` を使うため、Common Controls v6 を要求するマニフェストが必須です。  
これが無いと「TTM_ADDTOOL failed」というエラーでダイアログ作成に失敗します。

1. プロジェクト直下に `app.manifest` を配置する（内容は一般的な Common Controls v6 マニフェスト）。
2. そのディレクトリで以下を実行して `rsrc.syso` を生成する。

```bash
rsrc -manifest app.manifest -o rsrc.syso
```

`rsrc.syso` は **`main.go` と同じディレクトリに置くこと**。  
これで `go build` / `go run` / VSCode デバッグのいずれでもマニフェストが自動リンクされます。

### 3. 開発時のデバッグ方法

- **手順**
  1. `rsrc.syso` を作成済みであることを確認する（無ければ「2. マニフェストの有効化」を実行）。
  2. VSCode でこのフォルダを開く。
  3. すでに用意してある `.vscode/launch.json` の構成を選択して **F5 を押すだけ**。

裏側では Go 拡張機能が自動でビルド＆デバッグ用バイナリを作成し、  
`rsrc.syso` によってマニフェストが有効化されるため `TTM_ADDTOOL failed` は発生しません。

### 4. コードフォーマット (`go fmt`)

すべての Go ファイルを一括でフォーマットしたい場合は、プロジェクトルートで以下を実行します。

```bash
go fmt ./...
gofmt -w .
```

`go fmt ./...` はパッケージ単位でソースコードを整形し、`gofmt -w .` はカレントディレクトリ以下の `.go` ファイルをすべて書き換えます。

### 5. よくあるエラー

- **ダイアログの作成に失敗しました: TTM_ADDTOOL failed**
  - 原因: `rsrc.syso` が無い、または `app.manifest` が正しく埋め込まれていない。
  - 対処: `rsrc -manifest app.manifest -o rsrc.syso` を再実行し、プロジェクト直下に `rsrc.syso` があることを確認してからビルド/デバッグする。
